# 16.5 macOS 文件系统

FreeBSD 对 macOS 文件系统的支持主要通过 FUSE 实现。本节将介绍如何在 FreeBSD 上挂载 macOS 常用的文件系统。

## 目录结构

```sh
/
├── usr
│   ├── local
│   │   └── bin
│   │       └── apfs-fuse           # APFS FUSE 可执行文件
│   └── ports
│       └── filesystems
│           └── apfs-fuse           # apfs-fuse Ports 目录
├── etc
│   ├── rc.conf                     # 系统启动配置文件
│   └── fstab                       # 持久化挂载配置文件
├── dev
│   └── da0                         # 磁盘设备
└── media
    └── macOS                       # macOS 文件系统挂载点
```

## APFS 文件系统

Apple 文件系统（APFS）是 Apple 开发的新一代文件系统，自 macOS High Sierra 起成为 macOS 的默认文件系统。在 FreeBSD 上，可通过 `apfs-fuse` 实现对 APFS 分区的只读访问。

### 安装 apfs-fuse

使用 pkg 安装 apfs-fuse 软件包：

```sh
# pkg install apfs-fuse
```

使用 Ports 安装：

```sh
# cd /usr/ports/filesystems/apfs-fuse/
# make install clean
```

### 加载 fusefs 内核模块

在使用 FUSE 文件系统前，需加载 fusefs(4) 内核模块：

```sh
# kldload fusefs
```

使用 sysrc(8) 设置开机自动加载该模块：

```sh
# sysrc kld_list+=fusefs
```

### 识别 APFS 分区

将包含 APFS 分区的磁盘连接到计算机后，可使用 `gpart` 查看分区信息。APFS 分区通常显示为 `apple-apfs` 类型：

```sh
# gpart show -p da0  # 显示磁盘 da0 的分区信息
=>      40  488397088    da0  GPT  (233G)
        40     409600  da0p1  efi  (200M)
    409640  487987488  da0p2  apple-apfs  (233G)
```

### 挂载 APFS 分区

使用 `apfs-fuse` 挂载 APFS 分区。首先创建挂载目录：

```sh
# mkdir -p /media/macOS
```

将 APFS 分区 `/dev/da0p2` 挂载到 `/media/macOS`：

```sh
# apfs-fuse /dev/da0p2 /media/macOS
```

验证挂载情况并查看内容：

```sh
# ls /media/macOS
```

> **注意**
>
> 当前 `apfs-fuse` 主要提供只读支持，无法进行写入操作。

## HFS+ 文件系统

HFS+（分层文件系统加）是 macOS 在 APFS 之前使用的默认文件系统。FreeBSD 对 HFS+ 的支持同样通过 FUSE 实现。

### 安装 hfsfuse

使用 pkg 安装 hfsfuse 软件包：

```sh
# pkg install hfsfuse
```

使用 Ports 安装：

```sh
# cd /usr/ports/filesystems/hfsfuse/
# make install clean
```

### 识别 HFS+ 分区

HFS+ 分区通常显示为 `apple-hfs` 类型：

```sh
# gpart show -p da0
=>      40  488397088    da0  GPT  (233G)
        40     409600  da0p1  efi  (200M)
    409640  487987488  da0p2  apple-hfs  (233G)
```

### 挂载 HFS+ 分区

创建挂载目录并挂载 HFS+ 分区：

```sh
# mkdir -p /media/macOS
# hfsfuse /dev/da0p2 /media/macOS
```

验证挂载：

```sh
# ls /media/macOS
```

## 持久化挂载配置

如需在系统启动时自动挂载 macOS 文件系统，可在 `/etc/fstab` 中添加配置。对 FUSE 文件系统，建议使用 `late` 选项避免启动问题。

### APFS 持久化配置示例

```ini
/dev/da0p2  /media/macOS  fuse  rw,mountprog=/usr/local/bin/apfs-fuse,late  0  0
```

### HFS+ 持久化配置示例

```ini
/dev/da0p2  /media/macOS  fuse  rw,mountprog=/usr/local/bin/hfsfuse,late  0  0
```
