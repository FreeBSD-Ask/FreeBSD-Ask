# 17.6 Blocklistd

Blocklistd 可动态封禁与释放端口，防范 DoS 攻击行为，曾用名 Blacklistd。

## 启用 Blocklistd

要在系统启动时启用该守护进程，执行以下命令：

```sh
# service blocklistd enable
```

要手动启动服务，运行以下命令：

```sh
# service blocklistd start
```

## 创建 Blocklistd 规则集

Blocklistd 规则集位于 **/etc/blocklistd.conf** 文件，每行各填写一条规则。每条规则由空格或制表符分隔的多个字段组成。规则分为 `local` 和 `remote`，分别用于 Blocklistd 所在的本机和外部来源。

### 本地规则

默认配置文件 **/etc/blocklistd.conf** 内容如下：

```ini
# Blocklist rule
# adr/mask:port	type	proto	owner		name	nfail	duration
[local]
ssh		stream	*	*		*	3	24h
ftp		stream	*	*		*	3	24h
smtp		stream	*	*		*	3	24h
submission	stream	*	*		*	3	24h
#6161		stream	tcp6	christos	*	2	10m
*		*	*	*		*	3	60

# adr/mask:port	type	proto	owner		name	nfail	duration
[remote]
#129.168.0.0/16	*	*	*		=	*	*
#[2001:db8::]/32:ssh	*	*	*		=	*	*
#6161		=	=	=		=/24	=	=
#*		stream	tcp	*		=	=	=
```

文件 **/etc/blocklistd.conf** 中的本地规则如下所示：

```ini
[local]
ssh		stream	*	*		*	3	24h
ftp		stream	*	*		*	3	24h
smtp		stream	*	*		*	3	24h
submission	stream	*	*		*	3	24h
#6161		stream	tcp6	christos	*	2	10m
*		*	*	*		*	3	60
```

所有跟随 `[local]` 部分的规则都视为本地规则（这是默认设置），适用于本地机器。当遇到 `[remote]` 部分时，后续的规则将作为远程机器规则处理。通配符由星号 (`*`) 表示，匹配该字段中的任何内容。

七个字段各定义一条规则，字段之间以制表符或空格分隔：

- `adr/mask:port`、`type`、`proto`、`owner`：前四个字段用于标识需列入黑名单的流量。
  - `adr/mask:port`：首字段定义地址。在本地规则中，首字段是网络端口。
    
	`adr/mask:port` 地址字段的语法：

    ```sh
    [地址|接口][/掩码][:端口]
    ```
  
    `adr/mask:port` 地址字段可以按数字格式指定为 IPv4 或 IPv6（以方括号表示）；也可以使用接口名称，如 `em0`。

  - `type` 字段：定义套接字类型。TCP 套接字为 `stream` 类型，而 UDP 则用 `dgram` 表示。上面的示例使用的是 TCP，因为 SSH 使用的是 TCP 协议。
  - `proto` 字段：blocklistd 规则中可用的协议。可使用以下协议：`tcp`、`udp`、`tcp6`、`udp6` 或数字。如上例所示，通常使用通配符来匹配所有协议，除非有特别理由需要按特定协议区分流量。
  - `owner` 字段：定义报告事件的守护进程的有效用户或所有者。这里可以使用用户名或 UID，也可以使用通配符（参见上面的示例规则）。
- `name`、`nfail`、`duration`：接下来的三个字段定义 blocklistd 的行为。
   - `name` 字段：声明数据包过滤规则的名称，标志着规则的行为部分。默认情况下，blocklistd 将所有阻塞规则放置在 **/etc/pf.conf** 文件中的锚点 `blocklistd` 下，如下所示：

     ```sh
     anchor "blocklistd/*" in on $ext_if
     block in
     pass out
     ```

     对于单独的黑名单，可以在该字段中使用锚点名称。在其他情况下，使用通配符就足够了。当名称以连字符开头（`-ssh`）时，表示应使用带有默认规则名称的锚点。
	 
	 下面是修改后的示例，使用了连字符（`-`）：

     ```sh
     ssh             stream  *       *               -ssh       3       24h
     ```

     在这种规则下，新的黑名单规则将添加到锚点 `blocklistd-ssh` 中。

	 要因单个规则违规封锁整个子网，可以在规则名称中使用 `/`。这会使名称中 `/` 后面的部分被解释为应应用到规则中指定地址的掩码。例如，以下这条规则会封锁与指定地址相邻的所有 `/24` 网段的地址。

     ```sh
     22              stream  tcp       *               */24    3       24h
     ```

     > **注意**
     >
     > 这里需要指定正确的协议。IPv4 和 IPv6 对 `/24` 的处理不同，因此在此规则的第三个字段中不能使用 `*`。

     此规则定义：如果该网络中的任何主机行为不当，整个网络中的其他主机都将受到阻止。

   - `nfail` 字段：设置需多少次登录失败方可将远程 IP 列入黑名单。若在此位置使用通配符，则永不触发封禁。在上例中，定义了三次登录失败的限制，即在 SSH 上三次登录失败后，该 IP 将被封禁。

   - `duration` 字段：指定主机列入黑名单的时间。默认单位为秒，也可以指定 `m`、`h`、`d` 等后缀，分别表示分钟、小时和天。

完整的示例规则 `ssh		stream	*	*		*	3	24h` 表示，当 SSH 验证失败三次后，该主机将生成一条新的 PF 封锁规则。规则匹配时，首先按顺序检查 `local` 规则，从最具体（匹配条件最严格、范围最小的规则）到最不具体（匹配条件宽泛、范围大的规则）。匹配成功后，会应用 `remote` 规则，并由匹配的 `remote` 规则修改字段 `name`、`nfail` 和 `duration`。

### 远程规则

`remote` 规则用于指定 Blocklistd 如何根据当前被评估的远程主机调整其行为。`remote` 规则中的每个字段与 `local` 规则相同，唯一的区别在于 Blocklistd 使用这些字段的方式不同。以下用示例规则说明：


```sh
# adr/mask:port	type	proto	owner		name	nfail	duration
[remote]
#129.168.0.0/16	*	*	*		=	*	*
#[2001:db8::]/32:ssh	*	*	*		=	*	*
#6161		=	=	=		=/24	=	=
#*		stream	tcp	*		=	=	=
```

地址字段可以是 IP 地址（IPv4 或 IPv6）、端口，或者两者兼有。这样可以为特定的远程地址范围设置特殊规则，如本例所示。套接字类型、协议和所有者字段的解释与 `local` 规则完全相同。

name 字段有所不同：在 `remote` 规则中，等号（`=`）表示 blocklistd 使用匹配的 `local` 规则中的值。也就是说，防火墙规则条目会沿用匹配的 `local` 规则中的值，并加上 `/25` 前缀（掩码为 `255.255.255.128`）。当该地址范围内的连接被封锁时，整个子网都会受到影响。此处也可以使用 PF 锚点名称，此时 blocklistd 会将该地址块的规则添加到该锚点下。如果使用通配符，则使用默认表格。

`nfail` 列可以为某个地址定义自定义失败次数。这在设置特定规则的例外时很有用，比如允许某些用户更宽松的登录尝试机会。当第六个字段使用星号时，封锁功能即告禁用。

与来自本地网络（如办公室）的尝试相比，`remote` 规则可以对登录尝试施加更严格的限制。

## 配置 Blocklistd 客户端

在 FreeBSD 中，有一些软件包可以利用 Blocklistd 的功能。最常用的是 ftpd(8) 和 sshd(8)，它们可以阻止过多的连接尝试。要在 SSH 守护进程中启用 Blocklistd，请在 **/etc/ssh/sshd_config** 文件中添加以下行：

```sh
UseBlacklist yes
```

然后重新启动 sshd 使更改生效。

对于 ftpd(8)，可以使用 `-B` 参数来启用黑名单功能，可以在文件 **/etc/inetd.conf** 中设置，也可以作为标志在 **/etc/rc.conf** 中设置，如下所示：

```sh
ftpd_flags="-B"
```

以上即为让这些程序与 Blocklistd 通信所需的全部设置。

## 管理 Blocklistd

Blocklistd 提供了管理工具 blacklistctl(8)。用于查看通过 blocklistd.conf(5) 中定义的规则被列入黑名单的地址和网络。要查看当前被封禁的主机列表，可使用 `dump` 命令并搭配 `-b` 参数，如下所示：

```sh
# blacklistctl dump -b
      address/ma:port id      nfail   last access
213.0.123.128/25:22   OK      6/3     2019/06/08 14:30:19
```

此示例显示，从地址范围 `213.0.123.128/25` 发起的端口 `22` 登录尝试中，已发生 6 次，而允许的尝试次数为 3 次。显示的尝试次数超过允许次数，是因为 SSH 允许客户端在单个 TCP 连接上进行多次登录尝试。Blocklistd 不会中断当前正在进行的连接。输出中的“last access”列显示了最后一次连接尝试的时间。

要查看该主机在封锁列表上剩余的时间，可在之前的命令中添加 `-r` 参数。

```sh
# blacklistctl dump -br
      address/ma:port id      nfail   remaining time
213.0.123.128/25:22   OK      6/3     36s
```

在这个示例中，距离该主机解除封锁还有 36 秒。

## 从 Blocklistd 中移除主机

有时需要在剩余时间到期前将主机从封锁列表中移除。但 Blocklistd 本身没有提供这种功能。不过，可以使用 pfctl 从 PF 表中移除该地址。对于每个被封锁的端口，在 **/etc/pf.conf** 中 Blocklistd 锚点下都有一个子锚点。例如，用于封锁端口 22 的子锚点叫做 `blocklistd/22`。该子锚点内有一个表格，包含所有被封锁的地址。这个表格名为 port 加上端口号，在此例中为 `port22`。有了这些信息，就可以使用 pfctl(8) 来显示列出的所有地址：


```sh
# pfctl -a blocklistd/22 -t port22 -T show
...
213.0.123.128/25
...
```

在从列表中识别出需要解除封锁的地址后，使用以下命令将其从列表中移除：

```sh
# pfctl -a blocklistd/22 -t port22 -T delete 213.0.123.128/25
```

该地址已从 PF 中移除，但仍会出现在 blocklistctl list 中，因为 blocklistctl 并不知晓 PF 中所做的任何更改。blocklistd 数据库中的该条目最终会过期并从输出中删除。如果该主机再次匹配 blocklistd 中的某条封锁规则，该条目将会被重新添加。
