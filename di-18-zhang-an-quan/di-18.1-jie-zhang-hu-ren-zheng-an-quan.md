# 18.1 账户认证安全

在 FreeBSD 中维护安全的账户，对数据机密性、系统完整性和权限分离至关重要。它能防止未经授权的访问、恶意软件和数据泄露，同时确保合规并维护组织的声誉。

## 阻止登录

保护系统的第一步，通常是对账户进行审计。禁用所有无需登录访问的账户。

>**思考题**
>
>>确保 `root` 拥有一个强密码，并应定期轮替密码，且不应共享该密码。
>
>这种说法很流行但这一做法背后存在一定的安全隐患。
>
> - 强密码的复杂性使得其难以记忆，进而需要依赖不安全的存储方式，如纸质记录或文本文件。这种存储方式可能带来潜在的安全风险。
> - 定期更换密码进一步增加了密码管理的复杂性，而“密码管理工具”本身亦面临被信任的问题。
> - 尽管基于密钥的认证方式提供了一定的安全性，但其密钥丢失的风险使得身份验证过程更加复杂。
> - 此外，多因素认证（MFA）虽能增强安全性，但其依赖的设备一旦丢失或程序卸载，将造成类似的问题。

拒绝账户登录访问的手段有两种。

第一种是锁定账户。以下示例展示如何锁定 `imani` 账户：

```sh
# pw lock imani
```

第二种方法是将 Shell 改为 `/usr/sbin/nologin`，从而阻止登录访问。当用户尝试登录时，nologin  Shell 会阻止系统为其分配 Shell。

只有超级用户有权更改其他用户的 Shell：

```
# chsh -s /usr/sbin/nologin imani
```

### 16.3.2. 密码哈希

密码是技术领域必不可少的负担。既然必须使用，密码就应当足够复杂，并借助强大的哈希机制加密存储于密码数据库中的版本。FreeBSD 的 `crypt()` 库支持多种算法，包括 SHA256、SHA512 和 Blowfish，详见 [crypt(3)](https://man.freebsd.org/cgi/man.cgi?query=crypt&sektion=3&format=html)。

默认的 SHA512 不应降级为更弱的哈希算法，但可以升级为更安全的 Blowfish。

|  | Blowfish 不属于 AES，也不被视为符合任何联邦信息处理标准（FIPS）。在某些环境中可能不被允许使用。 |
| -- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

若要确认某位用户密码所使用的哈希算法，超级用户可以查看 FreeBSD 密码数据库中对应哈希值。每个哈希值均以一个符号开头，表明用于加密该密码的哈希机制类型。

DES 无起始符号。MD5 的符号为 `$`。SHA256 和 SHA512 的符号为 `$6$`。Blowfish 的符号为 `$2a$`。本例中，`imani` 的密码以 `$6$` 开头，说明其使用了默认的 SHA512 算法。注意，密码数据库中存放的是加密后的哈希值，而非原始密码：

```
# grep imani /etc/master.passwd
```

输出应类似于以下内容：

```text
imani:$6$pzIjSvCAn.PBYQBA$PXpSeWPx3g5kscj3IMiM7tUEUSPmGexxta.8Lt9TGSi2lNQqYGKszsBPuGME0:1001:1001::0:0:imani:/usr/home/imani:/bin/sh
```

哈希机制在用户的登录类别中设置。

可以运行以下命令来检查当前正在使用的哈希机制：

```sh
% grep passwd_format /etc/login.conf
```

输出应类似于以下内容：

```ini
:passwd_format=sha512:\
```

例如，要切换为 Blowfish，可将该行修改为：

```ini
:passwd_format=blf:\
```

接着，必须执行 [cap_mkdb(1)](https://man.freebsd.org/cgi/man.cgi?query=cap_mkdb&sektion=1&format=html) 以更新 login.conf 数据库：

```sh
# cap_mkdb /etc/login.conf
```

注意，此更改不会影响已有密码哈希。也就是说，所有用户都需要运行 `passwd` 修改密码，才能让新算法生效。

### 16.3.3. 密码策略执行

为本地账户实施强密码策略，是系统安全的一项基本配置。FreeBSD 可借助内置的可插拔认证模块（PAM）控制密码长度、强度与复杂度。

本节演示如何利用 [pam_passwdqc(8)](https://man.freebsd.org/cgi/man.cgi?query=pam_passwdqc&sektion=8&format=html) 模块设置密码的最短与最长长度，并强制要求混合字符。该模块在用户修改密码时生效。

配置前，先切换为超级用户，再将 /etc/pam.d/passwd 中包含 `pam_passwdqc.so` 的那一行取消注释。

然后，按密码策略编辑该行：

```ini
password        requisite       pam_passwdqc.so         min=disabled,disabled,disabled,12,10 similar=deny retry=3 enforce=users
```

参数含义见 [pam_passwdqc(8)](https://man.freebsd.org/cgi/man.cgi?query=pam_passwdqc&sektion=8&format=html)。

保存文件后，用户修改密码时会看到类似下面的提示：

```sh
% passwd
```

输出应类似于以下内容：

```text
Changing local password for user
Old Password:

You can now choose the new password.
A valid password should be a mix of upper and lower case letters,
digits and other characters.  You can use a 12 character long
password with characters from at least 3 of these 4 classes, or
a 10 character long password containing characters from all the
classes.  Characters that form a common pattern are discarded by
the check.
Alternatively, if no one else can see your terminal now, you can
pick this as your password: "trait-useful&knob".
Enter new password:
```

若输入的密码不符合策略，系统会拒绝并给出警告，用户可在配置的重试次数内反复尝试。

若组织要求密码定期过期，FreeBSD 可在 /etc/login.conf 的用户登录类别中设置 `passwordtime` 参数。

`default` 登录类别中包含一个示例：

```ini
#       :passwordtime=90d:\
```

因此，若想为此登录类别设定 90 天过期，只需移除注释符号（#），保存文件并执行：

```sh
# cap_mkdb /etc/login.conf
```

若要为单个用户设置过期时间，可向 `pw` 同时传递过期日期（或距离过期的天数）和用户名：

```sh
# pw usermod -p 30-apr-2025 -n user
```

如这里所示，过期日期以日、月、年的格式设置。更多信息请参阅 [pw(8)](https://man.freebsd.org/cgi/man.cgi?query=pw&sektion=8&format=html)。
