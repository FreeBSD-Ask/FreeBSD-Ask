# 18.2 资源限制

FreeBSD 提供 `/etc/login.conf` 静态资源限制与 `rctl` 动态资源限制两种机制。本节分别给出登录类别（login class）配置方法和进程级约束规则定义。

## 资源限制

在 FreeBSD 中，资源限制是指控制和管理各种系统资源分配给进程和用户的机制。这些限制旨在防止单个进程或用户消耗过多的资源，这可能导致性能下降或系统不稳定。资源限制有助于确保系统上所有活动进程和用户之间的公平资源分配。

FreeBSD 提供多种方法供管理员限制个人可使用的系统资源量。

传统方法是通过编辑 `/etc/login.conf` 文件来定义登录类别。虽然该方法仍受支持，但任何更改都需经过多个步骤：编辑该文件、重建资源数据库、对 `/etc/master.passwd` 文件进行必要的修改，以及重建密码数据库。这一过程可能非常耗时，取决于需要配置的用户数量。

`rctl` 提供了一种更细粒度的资源限制机制，它还可对进程和 jail 设置资源约束。

本节演示了两种控制资源的方法，从传统方法开始。

### 资源类型

FreeBSD 为多种类型的资源提供了限制，包括：

| 类型 | 描述 |
| ---- | ---- |
| CPU Time | 限制进程可消耗的 CPU 时间量 |
| Memory | 控制进程可使用的物理内存量 |
| Open Files | 限制进程可同时打开的文件数量 |
| Processes | 控制用户或进程可创建的进程数量 |
| File Size | 限制进程可创建的文件的最大大小 |
| Core Dumps | 控制是否允许进程生成核心转储文件 |
| Network Resources | 限制进程可使用的网络资源（例如套接字）数量 |

### 配置登录类别

在传统方法中，登录类别及其关联的资源限制在 `/etc/login.conf` 文件中定义。每个用户账户可被分配到某个登录类别，`default` 为默认类别。每个登录类别都关联一组登录能力（login capability），其形式为 `名称=值` 对，其中 *名称* 是已知标识符，*值* 为任意字符串，按对应 *名称* 的规则处理。

配置资源限制的第一步是打开并编辑 `/etc/login.conf` 文件，然后定位到要修改的用户类别所在节。

本例假设要使用的用户类别名为 `limited`，若不存在则创建之。

```ini
limited:\	# 用户类别名称 
        :maxproc=50:\ # 将 `limited` 类别下用户的最大进程数（maxproc）设为 50
        :tc=default: # 此用户类别继承 `default` 类别的默认设置
```

修改 `/etc/login.conf` 文件后，运行 `cap_mkdb` 生成 FreeBSD 用于应用这些设置的数据库：

```sh
# cap_mkdb /etc/login.conf
```

`chpass` 可为目标用户更改类别。执行：

```sh
# chpass username
```

这将打开文本编辑器，按如下方式添加新的 `limited` 类别：

```ini
#Changing user information for username.
Login: username
Password: $6$2H.419USdGaiJeqK$6kgcTnDadasdasd3YnlNZsOni5AMymibkAfRCPirc7ZFjjv
DVsKyXx26daabdfqSdasdsmL/ZMUpdHiO0
Uid [#]: 1001
Gid [# or name]: 1001
Change [month day year]:
Expire [month day year]:
Class: limited
Home directory: /home/username
Shell: /bin/sh
Full Name: User &
Office Location:
Office Phone:
Home Phone:
Other information:
```

现在，分配到 `limited` 类别的用户将具有最大 50 个进程的限制。注意，这仅是使用 `/etc/login.conf` 文件设置资源限制的一个示例。

修改 `/etc/login.conf` 文件后，用户需注销并重新登录方能使更改生效。

## 启用和配置资源限制

rctl 命令提供了一种控制资源限制的方法。它支持动态地将资源限制分配给特定进程或用户，与其所属的用户类别无关。

使用 rctl 前需先将其启用，在 `/boot/loader.conf` 文件中添加以下行并重启系统：

```sh
kern.racct.enable=1
```

然后，通过执行以下命令启用并启动 rctl 服务：

```sh
# sysrc rctl_enable="YES"
# service rctl start
```

此后即可使用 rctl 为系统设置规则。

规则语法（rctl.conf）由主语（subject）、主语 ID（subject-id）、资源（resource）和动作（action）共同控制，示例如下：

```text
subject:subject-id:resource:action=amount/per
```

例如，要约束某用户最多只能创建 10 个进程，执行：

```sh
# rctl -a user:username:maxproc:deny=10/user
```

要检查已应用的资源限制，执行 rctl：

```sh
# rctl
```

输出应类似于以下内容：

```text
user:username:maxproc:deny=10
```

如果将规则写入 `/etc/rctl.conf` 文件中，它们将在重启后继续生效。格式为单条规则，不带前导命令。例如，前述规则可添加为：

```ini
user:username:maxproc:deny=10
```

## 课后习题

1. 在 `/etc/login.conf` 文件中为一个测试用户类设置 `maxproc` 为 50，然后编写一个简单的 Shell 脚本在该类用户下创建超过 50 个子进程，观察系统行为并分析内核是通过何种机制阻止超限的。

2. 使用 `limits -a` 查看当前 Shell 的所有资源限制，将这些限制与 `sysctl kern.maxproc` 的输出进行对比，指出用户级限制与系统全局上限之间的关系。

3. 已知 fork 炸弹 `:(){ :|:& };:` 可能拖垮未做限制的系统。设计一个实验：如何防止这种问题。
