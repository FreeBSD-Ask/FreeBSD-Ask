# 18.3 安全等级

安全等级是 FreeBSD 内核内建的安全机制，通过 `sysctl kern.securelevel` 在 -1 至 3 五个级别间切换。本节说明各级别的权限限制与 **/etc/rc.conf** 中 `kern_securelevel_enable` 配置方法。

## 概述

安全等级是内核内建的安全机制。当安全等级为正值时，内核会限制某些操作——包括 root 用户在内也不例外。

安全等级机制会限制以下功能：

- 取消设置某些文件标志，如 `schg`（系统不可变标志）。
- 通过 **/dev/mem** 和 **/dev/kmem** 写入内核内存。
- 加载内核模块。
- 更改防火墙规则。

## 安全等级定义

内核共有五个安全等级。任何超级用户进程都可以提升等级，但没有进程能够降低等级。

各安全等级的定义如下：

- `-1`：**永久不安全模式** —— 系统始终处于不安全模式。此为默认初始值。

- `0`：**不安全模式** —— 不可变和仅追加标志可被关闭。所有设备可依其权限进行读写。

- `1`：**安全模式** —— 系统不可变和系统仅追加标志无法关闭；已挂载文件系统所在的磁盘、**/dev/mem** 和 **/dev/kmem** 均不可打开以写入；若平台存在 **/dev/io**，则完全禁止打开；无法加载或卸载内核模块。禁止使用 sysctl `debug.kdb.enter` 进入内核调试器（除非 MAC(9) 策略授予访问权限，例如 `mac_ddb(4)`）；也无法通过 `debug.kdb.panic`、`debug.kdb.panic_str` 及其他 sysctl 强制触发 panic 或 trap。

- `2`：**高度安全模式** —— 在安全模式的基础上，磁盘（无论是否已挂载）均不可打开以写入，mount 操作除外。此等级能阻止通过卸载文件系统来篡改文件数据，但也意味着在多用户模式下无法运行 newfs。此外，内核时间变更被限制在不超过 1 秒的范围内；尝试超出此范围修改时间将记录日志消息 “Time adjustment clamped to +1 second”。

- `3`：**网络安全模式** —— 在高度安全模式的基础上，IP 数据包过滤规则（`ipfw(8)`、`ipfirewall(4)` 和 `pfctl(8)`）均无法修改，`dummynet(4)` 或 `pf(4)` 配置也无法调整。

在 FreeBSD 安全等级中，`永久不安全模式` 与 `不安全模式` 的关键差异在于安全程度的强弱。`永久不安全模式` 彻底解除一切安全限制，而 `不安全模式` 虽放宽了某些约束，但仍保留了相当程度的控制与安全。

根据 `init(8)` 的行为，如果系统初始安全等级为 0，`init` 会在首次进入多用户模式之前将安全等级提升至 1。由于安全等级只能提升不能降低，此后即使在返回单用户模式时，系统也将至少维持在等级 1。另外，在引导序列中过早将安全等级设置为大于 1 的值可能会阻止 `fsck(8)` 修复不一致的文件系统，因此建议在 **/etc/rc** 末尾（所有多用户启动操作完成后）再设置安全等级。

## 修改安全等级

要调整系统的安全等级，首先须通过以下命令激活 `kern_securelevel_enable`：

```sh
# sysrc kern_securelevel_enable="YES"
```

再将 `kern_securelevel` 设为期望的安全等级（允许的取值范围为 -1 至 3）：

```sh
# sysrc kern_securelevel=2
```

重启后安全级别将生效。

若要在正在运行的系统中查看当前安全等级状态，可执行：

```sh
# sysctl -n kern.securelevel
2
```

输出包含安全等级的当前值。若该值大于 0，则表明系统至少已启用部分安全保护。
