# 17.2 Packet Filter（PF）

PF（Packet Filter，包过滤器）是一款源自 OpenBSD 的防火墙，具备 ALTQ（Alternate Queuing，交替队列）等多种功能。ALTQ 是 PF 的流量队列管理机制，实现服务质量（QoS）控制。

## 启用 PF

启用 PF 前，需先加载内核模块、准备配置文件并启动相关服务，具体步骤如下。

```sh
# kldload pf                  # 加载 pf 内核模块，使系统能够识别和使用 PF 防火墙
# cp /usr/share/examples/pf/pf.conf /etc/  # 复制示例文件作为默认配置规则集，否则 pf 无法启动
# service pf enable           # 设置 pf 在系统启动时自动启动
# service pf start            # 启动 pf 服务
```

- ① 如果不执行上述操作，则会提示 `pfctl: /dev/pf: No such file or directory`，此时可重启系统后再执行 `service pf start`。

- ② 若未复制配置文件，将提示如下信息：

```sh
/etc/rc.d/pf: WARNING: /etc/pf.conf is not readable.
```

PF 的管理命令为 `pfctl`。

## 文件结构

PF 防火墙的配置文件结构如下。

```sh
/
├── etc
│   └── pf.conf
└── usr
    └── share
        └── examples
            └── pf
                └── pf.conf
```

## PF syncookie 防护

SYN 洪水攻击是常见的拒绝服务攻击方式，攻击者发送大量伪造源地址的 SYN 包，使服务器耗尽资源维持半开连接。PF 防火墙通过 syncookie 机制防护此类攻击。

syncookie 的工作原理是：当检测到 SYN 洪水时，服务器不再为半开连接分配资源，而是将连接状态信息编码至 SYN-ACK 的初始序列号返回给客户端。当客户端回复合法的 ACK 时，服务器从该序列号还原连接状态并建立连接。这样，服务器无需为半开连接维护状态表，从而有效抵御 SYN 洪水攻击。

在 PF 配置中，以 `synproxy` 状态机制实现 SYN 代理防护：

```ini
pass in on $ext_if proto tcp from any to ($ext_if) port www synproxy state
```

`synproxy state` 会使 PF 代替服务器与客户端完成三次握手，只有握手成功后才将连接转发给服务器。
