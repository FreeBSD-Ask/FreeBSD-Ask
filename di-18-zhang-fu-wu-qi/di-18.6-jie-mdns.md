# 18.6 零配置网络（mDNS/DNS-SD）

零配置网络（[Zero-configuration networking](https://en.wikipedia.org/wiki/Zero-configuration_networking)，有时被称为 *Zeroconf*）是一组简化网络配置的技术，旨在让网络设备在不需要管理员参与配置的情况下，实现自动地址配置、域名解析和服务发现。

该功能在嵌入式设备上非常实用，在某些场景下无法通过屏幕或串口获得树莓派的 IP 地址，然而又需要获得 SSH 连接树莓派进行使用。在树莓派 OS 上，Avahi 服务开箱即用，且支持多播 DNS。在这种环境下，可以直接通过 `ping raspberrypi.local` 来获得树莓派的 IP 地址，或者直接使用 SSH 连接到 `raspberrypi.local`。

零配置网络的主要部分包括：

- 链路本地寻址（Link-Local Addressing），提供数值型网络地址的自动分配。
- 多播 DNS（*mDNS*），提供主机名的自动分发与解析。
- 基于 DNS 的服务发现（*DNS-SD*），提供服务实例的自动发现。

zeroconf 的流行实现是 [Avahi](https://avahi.org/)。而 **nss-mdns** 能在所有使用 **nsswitch**（The Name Service Switch (NSS), 名称切换服务）的工具中，通过 mDNS 协议查找 `*.local` 后缀的主机名。

## 安装 Avahi 和 nss-mdns

使用 pkg 安装：

```sh
# pkg install avahi-app nss_mdns
```

使用 Ports 安装：

```sh
# cd /usr/ports/net/avahi-app/ && make install clean
# cd /usr/ports/dns/nss_mdns/ && make install clean
```

## 配置名称切换服务

dns/nss_mdns 将安装以下 NSS 模块。

```sh
lib/nss_mdns.so.1
lib/nss_mdns_minimal.so.1
lib/nss_mdns4.so.1
lib/nss_mdns4_minimal.so.1
lib/nss_mdns6.so.1
lib/nss_mdns6_minimal.so.1
lib/nss_mdns.so
lib/nss_mdns_minimal.so
lib/nss_mdns4.so
lib/nss_mdns4_minimal.so
lib/nss_mdns6.so
lib/nss_mdns6_minimal.so
```

关于上述模块的说明，参见 [avahi/nss-mdns](https://github.com/avahi/nss-mdns)。

`/etc/nsswitch.conf` 文件是 nsswitch 的配置文件，源代码位于 `lib/libc/net/nsswitch.conf`，默认文件内容如下：

```ini
#
# nsswitch.conf(5) - name service switch configuration file
#
group: compat
group_compat: nis
hosts: files dns
netgroup: compat
networks: files
passwd: compat
passwd_compat: nis
shells: files
services: compat
services_compat: nis
protocols: files
rpc: files
```

为了启用 mDNS，需要在 `hosts: files dns` 一行后面追加 `mdns` 和 `mdns6`。

```ini
hosts: files dns mdns mdns6
```

## 启用服务

D-Bus 是作为 Avahi 的依赖自动安装的，设置为开机自启：

```sh
# service dbus enable
```

现在启动 D-Bus 服务：

```sh
# service dbus start
```

将 Avahi 守护进程设置为开机自启：

```sh
# service avahi-daemon enable
```

现在启动 Avahi 服务：

```sh
# service avahi-daemon start
```
