# 19.8 ZFS 调优

## 技术潜能与现实困境

ZFS 的性能优势与高级特性需要针对性的参数调优才能充分发挥，调优策略依赖具体环境，需结合存储硬件、工作负载特征与使用场景作个性化配置，主要调优方向包括 ARC 缓存大小、记录大小、压缩算法选择等。ZFS 不属于典型的开箱即用型文件系统。

## 调优

调整可调参数，使 ZFS 在不同的工作负载下表现最佳。

- `vfs.zfs.arc_max` - ARC 的最大值。默认值是减去 1 GB 后的所有空余 RAM，或者是所有 RAM 的 5/8，以较大的值为准。此值不得低于 64 MiB。如果系统运行其他守护进程或进程可能需要内存，可以使用较小的值。可以在运行时通过 sysctl(8) 调整此值，或在 **/boot/loader.conf** 或 **/etc/sysctl.conf** 中设置。

  > **注意** 
  >
  > 运行时不可将此值设回 `0`；将此值减小到当前 ARC 大小以下时，不会在没有内存压力的情况下导致 ARC 立即收缩。
- `vfs.zfs.arc_min` - ARC 的最小值。默认值取 32 MiB 与总 RAM 的 1/32 中的较大值。可以调整此值，以防止其他应用程序挤出整个 ARC。可以在运行时通过 sysctl(8) 调整此值，并在 **/boot/loader.conf** 或 **/etc/sysctl.conf** 中设置。
- `vfs.zfs.vdev_min_auto_ashift` - 池创建时自动使用的最低 `ashift`（扇区大小）。该值是 2 的幂。默认值为 `9`，表示 `2^9 = 512`，即 512 字节的扇区大小。为了避免 **写放大** 并获得最佳性能，应将此值设置为池中设备使用的最大扇区大小。 常见的驱动器有 4 KB 扇区。使用默认的 `ashift` 值 `9` 与这些驱动器会导致写放大。在这些设备中，单个 4 KB 写入的数据将以八次 512 字节写入的方式写入。ZFS 尝试在创建池时从所有设备读取原生扇区大小，但具有 4 KB 扇区的驱动器为兼容性考虑报告其扇区为 512 字节。在创建池之前将 `vfs.zfs.vdev_min_auto_ashift` 设为 `12`（`2^12 = 4096`），即可强制 ZFS 使用 4 KB 块，从而获得最佳性能。

  强制使用 4 KB 块在计划进行磁盘升级的池中也非常有用。目前的磁盘使用 4 KB 扇区，而 `ashift` 值在创建池后无法更改。

  在某些特定情况下，较小的 512 字节块大小可能更为合适。例如，在使用 512 字节磁盘用于数据库或虚拟机存储时，较小的块在小随机读取时会传输更少的数据。这可以在使用较小的 ZFS 记录大小时提供更好的性能。
- `vfs.zfs.prefetch_disable` - 禁用预取。值为 `0` 时启用，值为 `1` 时禁用。默认值为 `0`（启用预取）。预取会将比请求的块大得多的数据块读取到 ARC 中，预期稍后会用到这些数据。如果工作负载有大量随机读取，禁用预取可能因减少不必要的读取而提高性能。可以随时通过 sysctl(8) 调整此值。
- `vfs.zfs.l2arc_write_max` - 限制每个 L2ARC 设备每秒写入的最大数据量。默认值为 32 MiB。此可调参数限制写入设备的数据量，延长 SSD 的使用寿命。总 L2ARC 吞吐量随池中缓存设备数量线性增长。可以随时通过 sysctl(8) 调整此值。
- `vfs.zfs.txg_timeout` — 事务组（transaction group）之间的最大秒数。当前事务组写入池中，如果自上一个事务组以来经过了此时间，则启动新的事务组。如果写入了足够的数据，事务组可能会提前触发。默认值为 `5` 秒。较大的值可能因延迟异步写入而提高读取性能，但这可能导致写入事务组时性能不均匀。可以随时通过 sysctl(8) 调整此值。
- `vfs.zfs.arc_meta_balance`（默认值 500）来进一步控制元数据与数据在幽灵命中（ghost hit）时的动态平衡。`vfs.zfs.arc_meta_balance` 的值大于 100 时，通过按比例降低幽灵数据命中对目标比率的影响来增加元数据缓存。
- 现代的 L2ARC Turbo Warmup 机制由 `vfs.zfs.l2arc_feed_again`（默认值 1，启用）控制，当 L2ARC 为冷时以最快速度填充。
- `vfs.zfs.vdev_max_active`（默认值 1000），该参数限制每个设备的最大并发 I/O 操作数。
- `vfs.zfs.vdev_trim_on_init` — 控制是否在新添加的 vdev 上执行初始化 TRIM。此外，TRIM 操作的 I/O 并发数受 I/O 调度器的 `vfs.zfs.vdev_trim_min_active`（默认值 1）和 `vfs.zfs.vdev_trim_max_active`（默认值 2）控制。
- `vfs.zfs.top_maxinflight` — （默认值 32）。限制每个顶层 vdev 的最大并发 I/O 数。现代的 I/O 调度器为每个队列提供了 `min_active` 和 `max_active` 参数以实现更精细的控制。
- 以 tick 为单位设置 scrub 的每次 I/O 之间的延迟。`vfs.zfs.vdev_scrub_min_active`（默认值 1）和 `vfs.zfs.vdev_scrub_max_active`（默认值 2）。当 vdev 空闲时，scrub 的并发数自动提升至 `max_active`。
- `vfs.zfs.resilver_delay` —（默认值 2）。以 tick 为单位设置 resilver 的每次 I/O 之间的延迟。I/O 调度器中还有 `vfs.zfs.vdev_rebuild_min_active`（默认值 1）和 `vfs.zfs.vdev_rebuild_max_active`（默认值 3）可替代使用。
- `vfs.zfs.vdev_nia_delay`（默认值 5）和 `vfs.zfs.vdev_nia_credit`（默认值 5）控制：当没有交互式 I/O 活动且已完成 `vfs.zfs.vdev_nia_delay` 个非交互式操作后，vdev 被视为“空闲”，非交互式操作的并发数提升至 `max_active`。
