# 14.11 系统日志管理

## 系统日志概述

系统日志是 FreeBSD 系统中记录系统运行状态、错误信息、安全事件等重要信息的关键组成部分。通过系统日志，管理员可以了解系统的运行情况，诊断问题，以及监控安全事件。

FreeBSD 提供了完整的日志管理框架，包括日志收集、存储、轮转和分析功能。本章将介绍 FreeBSD 的系统日志管理机制和常用工具。

## syslog 系统日志服务

FreeBSD 使用 `syslog` 作为默认的系统日志服务，负责接收、处理和存储系统和应用程序的日志消息。`syslogd` 是 syslog 服务的守护进程，它通过 `/etc/syslog.conf` 配置文件来决定如何处理不同类型的日志消息。

### syslog.conf 配置文件

`/etc/syslog.conf` 是 syslog 服务的主要配置文件，它定义了日志消息的来源和目的地。每一行配置包含两个字段，中间用制表符分隔：

- **选择器（facility.level）**：指定要记录的日志来源和级别
- **动作（action）**：指定日志消息的处理方式

```ini
# 示例 syslog.conf 配置片段
*.err;kern.debug;auth.notice /dev/console
daemon.notice;auth.notice  /var/log/messages
*.notice;auth.info;kern.err /var/log/messages
mail.info                  /var/log/maillog
ftp.info                   /var/log/xferlog
cron.*                     /var/log/cron
```

### 日志设施（Facility）

日志设施表示日志消息的来源，常见的设施包括：

- `kern`：内核消息
- `user`：用户程序消息
- `mail`：邮件系统
- `daemon`：系统守护进程
- `auth`：安全/认证消息
- `syslog`：syslogd 内部消息
- `lpr`：打印系统
- `news`：网络新闻系统
- `uucp`：UUCP 系统
- `cron`：cron 守护进程
- `authpriv`：安全/认证消息（私有）
- `ftp`：FTP 守护进程
- `local0-local7`：本地使用设施

### 日志级别（Level）

日志级别表示消息的严重程度，从低到高依次为：

- `debug`：调试信息
- `info`：一般信息
- `notice`：正常但重要的信息
- `warning`：警告信息
- `err`：错误信息
- `crit`：严重错误
- `alert`：需要立即采取行动的情况
- `emerg`：紧急情况（系统不可用）

当指定一个级别时，会包含该级别及以上级别的所有消息。例如，`daemon.notice` 会记录 daemon 设施的 notice、warning、err、crit、alert 和 emerg 级别的消息。

### 重启 syslogd

修改 `syslog.conf` 后，需要重启 `syslogd` 服务以使配置生效：

```sh
# 重新加载 syslogd 配置（不重启进程）
# /etc/rc.d/syslogd reload

# 或者完整重启 syslogd 服务
# /etc/rc.d/syslogd restart
```

## 常见日志文件

FreeBSD 系统将日志文件存储在 `/var/log` 目录中。以下是一些常见的重要日志文件：

- `/var/log/messages`：系统主日志文件，包含大部分系统消息
- `/var/log/auth.log`：认证相关日志（SSH、登录等）
- `/var/log/maillog`：邮件系统日志
- `/var/log/cron`：cron 任务执行日志
- `/var/log/xferlog`：FTP 传输日志
- `/var/log/security`：安全相关日志
- `/var/log/dmesg.today`：当日内核启动消息
- `/var/log/dmesg.yesterday`：昨日内核启动消息

## newsyslog 日志轮转

随着时间推移，日志文件会不断增长。为了防止磁盘空间耗尽，FreeBSD 使用 `newsyslog` 来管理日志文件的轮转。`newsyslog` 会定期将当前日志文件重命名为备份文件，并创建新的日志文件。

### newsyslog.conf 配置

`newsyslog` 的配置文件是 `/etc/newsyslog.conf`，它定义了哪些日志文件需要轮转、轮转的频率以及保留多少个备份文件。

```ini
# 示例 newsyslog.conf 配置片段
# logfilename          [owner:group]    mode count size when  flags [/pid_file] [sig_num]
/var/log/messages                        644  5     100  *     Z
/var/log/auth.log                        600  7     100  *     Z
/var/log/maillog                         644  7     *    @T00  Z
/var/log/cron                            600  3     100  *     Z
```

配置字段说明：

- **logfilename**：日志文件路径
- **owner:group**：日志文件的所有者和组（可选）
- **mode**：日志文件的权限
- **count**：保留的备份文件数量
- **size**：触发轮转的文件大小（KB，* 表示不限制）
- **when**：轮转时间（* 表示不限，@T00 表示每天 0 点，$M0D0 表示每月第一天）
- **flags**：其他标志（Z 表示压缩备份，J 表示使用 xz 压缩，B 表示二进制日志）
