# 13.6 openSUSE 兼容层

openSUSE 兼容层基于 Leap dnf 容器镜像构建，opi 工具会自动安装 zypper 包管理器以实现双包管理器共存。本节提供自动化脚本及软件源配置指引。

## 基于 openSUSE Leap dnf 容器的 Shell 脚本

```sh
#!/bin/sh

ROOT_DIR=/compat
# 统一使用 zypper 命令
UPDATE_CMD="zypper --non-interactive refresh"
UPGRADE_CMD="zypper --non-interactive update"

echo "========================================"
echo "  Welcome to FreeBSD SUSE Installer"
echo "========================================"
echo "Select your SUSE distribution:"
echo "1) openSUSE Tumbleweed"
echo "2) openSUSE Leap 16.0"
echo "3) SUSE Linux Enterprise 16.0"
echo "----------------------------------------"
echo -n "Enter your choice (1-3): "
read DIST_CHOICE

case "$DIST_CHOICE" in
    1)
        DIST_Linux="tumbleweed"
        FILE="openSUSE-Tumbleweed-20260422.x86_64-2.97-Build2.97.wsl"
        URL="https://github.com/openSUSE/WSL-instarball/releases/download/v20260423.0"
        EXPECTED_HASH="c0dcf707276626e7a6227404c6a1e351dfdb9d39f2538d2b8a78a2c2e41f5912"
        ;;
    2)
        DIST_Linux="leap16"
        FILE="openSUSE-Leap-16.0-16.0.x86_64-22.57-Build22.57.wsl"
        URL="https://github.com/openSUSE/WSL-instarball/releases/download/v20251001.0"
        EXPECTED_HASH="0d1faa095153beee0a9b5089b0f9aa3d2aec95e2cdcffdeeff84dd54c48b8393"
        ;;
    3)
        DIST_Linux="sle16"
        FILE="SUSE-Linux-Enterprise-16.0-16.0.x86_64-1.74-Build1.74.wsl"
        URL="https://github.com/SUSE/WSL-instarball/releases/download/v20260515.0"
        EXPECTED_HASH="d98c6790a11ce149e37fe7ee276f8093f927a5882a674b1f1912a7f015311650"
        ;;
    *) echo "Invalid choice."; exit 1 ;;
esac

TARGET_PATH="${ROOT_DIR}/${DIST_Linux}"
mkdir -p "${TARGET_PATH}"

# 基础服务环境配置
service linux enable
service linux start
sysctl compat.linux.emul_path="${TARGET_PATH}"
# 确保持久化
grep -q "compat.linux.emul_path" /etc/sysctl.conf || echo "compat.linux.emul_path=${TARGET_PATH}" >> /etc/sysctl.conf

# 下载与校验
echo "Downloading and verifying image..."
fetch "${URL}/${FILE}"
if [ "$(sha256 -q "${FILE}")" != "$EXPECTED_HASH" ]; then
    echo "Checksum mismatch! Aborting."
    exit 1
fi

# 解压
tar xvpf ${FILE} -C ${TARGET_PATH} --numeric-owner 2>&1 | grep -v "Error exit delayed from previous errors"

# 网络与初始化
mkdir -p ${TARGET_PATH}/etc/
echo "nameserver 223.5.5.5" > ${TARGET_PATH}/etc/resolv.conf

echo "Updating system..."
chroot "${TARGET_PATH}" /bin/bash -c "$UPDATE_CMD && $UPGRADE_CMD"

rm -f "${FILE}"
echo "Installation complete. Access via: chroot ${TARGET_PATH} /bin/bash"
```

## 故障排除与未竟事宜

### 使用 LXC Leap 镜像时 zypper 不可用

具体而言，在使用 openSUSE Tumbleweed 时，dnf 和 zypper 均不可用。该问题不影响 Leap 对应的脚本，但影响 Tumbleweed 对应的脚本，导致无法正常运行。已提交 [Bug 报告](https://bugzilla.opensuse.org/show_bug.cgi?id=1213158)。

例如安装 Vim 时会报错：

```sh
Installation of perl-5.36.1-1.2.x86_64 failed:
Error Subprocess failed Error: RPM failed: Command exited with status 1
```

在 LXC Leap 中，用 rpm 替换 rpm-ndb 可解决该问题，且 dnf 不受影响，该问题仅存在于 zypper 中。然而，在 openSUSE Tumbleweed LXC 中，dnf 和 zypper 都存在该问题，替换 rpm-ndb 无法解决。
