# 31.1 文件传输协议（FTP）概述

文件传输协议（FTP）为用户提供文件上传与下载服务。

## 概述

FTP（File Transfer Protocol，文件传输协议）是 TCP/IP 协议栈中最早的应用层协议之一，用于网络文件传输。

在 FreeBSD 中搭建 FTP 服务器，可在局域网或互联网环境中快速传输与共享文件。

本节介绍三种常用的 FTP 服务器软件：

| 软件 | 特点 | 适用场景和人群 |
| ---- | ---- | -------------- |
| Pure-FTPd | ISC 许可证授权（与 2 条款 BSD 许可证功能等价）、安全优先、支持多种操作系统、FTP 协议规范实现较为完整 | 为 ISP 和托管服务提供高度灵活性，适用于新手 |
| ProFTPD | GPLv2 许可证授权、支持多种操作系统、单一主配置文件、可扩展性强 | 适合熟悉 Apache HTTP Server 的用户（语法类似） |
| vsftpd | GPLv2（含例外条款）许可证授权、支持 IPv6 和 SSL 加密、支持虚拟用户、虚拟 IP、细粒度用户与 IP 控制、带宽限制 | 面向生产环境 |

## FTP 协议的安全性

标准 FTP 协议的传输过程不安全：密码与数据均以明文发送，默认部署时不加密。

~~或许完全淘汰 FTP 协议才是唯一的解决方案。~~

> **注意**
>
> 运行匿名 FTP 服务器可能存在潜在风险。尤其应审慎评估是否允许匿名用户上传文件。FTP 站点可能成为未经授权的商业软件交易的场所，甚至可能引发更严重的后果。如果需要匿名 FTP 上传，请确保权限设置正确，以保证在管理员审查文件之前，其他匿名用户无法读取。

## FTP 客户端使用示例

使用 telnet 测试 FTP 服务：

```sh
# telnet localhost 21
Trying 127.0.0.1...
Connected to localhost.
Escape character is '^]'.
220 FTP Server ready
quit
221 Goodbye.
```

上述命令用于测试本地主机 FTP 服务的端口 21 是否可用。

使用 `ftp` 命令可便捷地连接至 FTP 服务器。其基本用法如下：

```sh
ftp [选项] [IP 地址]
```

FTP 命令：

```sh
put              上传
bell             在文件传输完成后发出提示音
dir/ls           显示当前目录中的文件及文件夹
cd               切换目录
delete           删除文件
get 远程文件     下载服务器上的远程文件
bye              结束与服务器的会话
```

此外，以下 FTP 协议命令可通过 `quote` 发送至服务器：

```sh
quote acct [账户信息]  提交账户信息（发送 ACCT 命令）
quote feat            显示该服务器支持的功能（发送 FEAT 命令）
```
