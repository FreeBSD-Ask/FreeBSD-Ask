# 25.7 Node.js 环境

Node.js 依赖特定版本的 **/lib/libcrypto.so**，在 FreeBSD 上安装前需先更新基本系统。本节涵盖 pkg 安装、版本兼容性说明及 npm 基础配置。

## 更新基本系统

在 FreeBSD 上配置 Node.js 环境前，需要先了解系统兼容性方面的注意事项。使用 pkg 安装的二进制 Node.js 依赖于特定版本的 **/lib/libcrypto.so.xxx** 文件，这意味着在 FreeBSD 上使用 Node.js 时，需要关注 FreeBSD 系统的版本，尤其是当配置 pkg 使用 latest 源时。

因此，在 FreeBSD 上安装 Node.js 前，应先更新基本系统，然后再进行安装。对于使用传统 Distribution Sets 安装的系统（FreeBSD 14 及更早版本，以及 FreeBSD 15 中选择传统方式的安装），使用 `freebsd-update fetch && freebsd-update install`；对于 FreeBSD 15.0 起采用 Pkgbase 方式安装的系统，基本系统通过 `pkg upgrade` 进行更新。

## Yarn + Node.js

使用 pkg 安装：

```sh
# freebsd-update fetch # 必须先更新基本系统
# freebsd-update install
# pkg install yarn             # 会自动安装对应版本的 nodejs
```

或使用 Ports 安装：

```sh
# cd /usr/ports/www/yarn/
# make install clean # 使用 ports 编译可规避上述问题
```

## NPM + Node.js

使用 pkg 安装：

```sh
# pkg install npm  # 安装 NPM 包管理器
```

或使用 ports 安装：

```sh
# cd /usr/ports/www/npm/
# make install clean # 使用 ports 编译可以规避上述问题
```

## FreeBSD NPM 切换软件源

设置 npm 使用国内镜像源以加快下载速度：

```sh
# npm config set registry https://registry.npmmirror.com
```

该地址为原淘宝镜像站，详情见 <https://npmmirror.com>，提供 NPM 国内镜像加速服务说明。

## 故障排除与未竟事宜

### ld-elf.so.1 错误

如果跳过了 FreeBSD 基本系统的更新而直接安装软件，那么在 FreeBSD 上，可能会遇到以下错误：

```sh
# pkg install yarn   # 安装 Yarn 包管理器
$ node              # 启动 Node.js 交互式命令行
ld-elf.so.1: /lib/libcrypto.so.111: version OPENSSL_1_1_1e required by /usr/local/bin/node not found
```

> **注意**
>
> 上述错误信息中的 `libcrypto.so.111` 对应 OpenSSL 1.1.1（FreeBSD 13 及更早版本）。在 FreeBSD 14 及 15 中，基本系统的 OpenSSL 已升级至 3.x，对应的库文件为 `libcrypto.so.8`，错误信息会有所不同，但根本原因相同——基本系统需要先更新。

如果使用 Ports 编译安装，上述问题不会出现，因为 Ports 会根据当前系统的库版本进行编译。
